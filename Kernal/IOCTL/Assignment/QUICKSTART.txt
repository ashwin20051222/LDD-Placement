 STEP-BY-STEP MANUAL SETUP:

   Step 1: Navigate to directory
   
   cd "/home/silicondoc/Documents/Class file/LDD/Kernal/IOCTL/Assignment"

   Step 2: Build the kernel module
   
   make clean
   make
   
    Should create: gpio_irq_driver.ko

   Step 3: Load the module
   
   sudo insmod gpio_irq_driver.ko

    Verify: lsmod | grep gpio

   Step 4: Check kernel logs
   
   dmesg | tail -5
   
    Look for: "GPIO IRQ ioctl driver loaded"
    Note the Major Number (e.g., 242)

   Step 5: Create device node
   
   MAJOR=$(grep gpioirqdev /proc/devices | awk '{print $1}')
   sudo mknod /dev/gpioirqdev c $MAJOR 0
   sudo chmod 666 /dev/gpioirqdev

    Verify: ls -la /dev/gpioirqdev

   Step 6: Compile user application
   
   gcc -o gpio_user_app gpio_user_app.c

   Step 7: Run the application
   
   ./gpio_user_app

    Should display: "Waiting for GPIO interrupts..."
    Waits for events (Press Ctrl+C to exit)

   Step 8: In another terminal, trigger events
   
   gcc -o gpio_test_trigger gpio_test_trigger.c
   ./gpio_test_trigger 5 1

    Triggers 5 events, 1 second apart
    First terminal shows: "[IRQ #1] GPIO INTERRUPT DETECTED!"

 TROUBLESHOOTING:

   Problem: "Module not found"
   Solution: Run "make" first to compile the module

   Problem: "Device not found: /dev/gpioirqdev"
   Solution: Run "make setup-auto" to create device node

   Problem: "Cannot open device"
   Solution: Check permissions with "ls -la /dev/gpioirqdev"

   Problem: "No interrupts received"
   Solution: Run gpio_test_trigger in another terminal

   Problem: Build error
   Solution: Install kernel headers: sudo apt install linux-headers-$(uname -r)
