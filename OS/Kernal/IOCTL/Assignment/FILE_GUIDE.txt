 SOURCE CODE FILES (Buildable C code)


gpio_irq_driver.c (4.5 KB)
 What: Kernel module source code
 Purpose: Main driver with ioctl, poll(), IRQ support
 Lines: ~200 lines
 Contains:
   gpio_ioctl() handler
   gpio_poll() handler
   gpio_irq_handler() for interrupts
   Module init/exit functions
   Thread-safe state management
 Compiles to: gpio_irq_driver.ko
 Build with: make

gpio_user_app.c (4.6 KB)
 What: User-space application
 Purpose: Monitor GPIO interrupts
 Lines: ~150 lines
 Features:
   Opens device /dev/gpioirqdev
   Sets GPIO pin and edge type
   Uses poll() to wait for events
   Calls ioctl() to control driver
   Displays interrupt statistics
 Run with: gcc -o gpio_user_app gpio_user_app.c && ./gpio_user_app
 Supports command-line arguments

gpio_test_trigger.c (2.7 KB)
 What: Test utility / interrupt simulator
 Purpose: Simulate GPIO events for testing
 Lines: ~90 lines
 Features:
   Simulates multiple interrupt events
   Configurable event count and delay
   Works with gpio_user_app
   Useful for testing without hardware
 Compile: gcc -o gpio_test_trigger gpio_test_trigger.c
 Run: ./gpio_test_trigger 5 1  (5 events, 1 sec delay)
 Use in second terminal while gpio_user_app runs

Makefile (1.5 KB)
 What: Build automation
 Targets:
   make                 - Compile the module
   make clean          - Remove build files
   make setup-auto     - Create device node
   make dmesg-follow   - Follow kernel logs
   make install        - Show install command
   make uninstall      - Show uninstall command
   make help           - Show all targets
 Auto-detects kernel version
 Provides helper functions for device setup
